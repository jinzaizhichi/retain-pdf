## 接口地址

主任务事件流：

`GET /api/v1/jobs/{job_id}/events?limit=50&offset=0`

OCR 子任务事件流：

`GET /api/v1/ocr/jobs/{job_id}/events?limit=50&offset=0`

返回包装固定为：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "items": [],
    "limit": 50,
    "offset": 0
  }
}
```

前端读取路径是 `data.items`，不是顶层 `items`。

## 当前口径

`/events` 返回的是 Rust canonical event：

- `stage`：公共阶段，面向前端展示。
- `substage`：机器可读子阶段。
- `lane`：`main | background | artifact | diagnostic`。
- `event_type`：`progress | artifact | terminal | error | diagnostic`。
- `progress`：公共进度对象。
- `message`：人类可读日志，不能作为机器判断依据。

旧文档中提到的 `progress_current` / `progress_total` / `progress_unit` 是内部来源字段。新前端优先读 `progress`。
