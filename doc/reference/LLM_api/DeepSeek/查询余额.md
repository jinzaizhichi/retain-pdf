# DeepSeek 查询余额

DeepSeek 官方余额查询接口用于检查当前 API 账号是否还有可调用余额。RetainPDF 如果要在前端或后端做 DeepSeek Key 检测，可以用这个接口补充判断 `402 Insufficient Balance` 这类问题。

## 接口

```http
GET https://api.deepseek.com/user/balance
Authorization: Bearer <DEEPSEEK_API_KEY>
Accept: application/json
```

注意：这个接口不带 `/v1`。项目里的聊天接口默认 `base_url` 是 `https://api.deepseek.com/v1`，但余额接口应调用 `https://api.deepseek.com/user/balance`，不要拼成 `https://api.deepseek.com/v1/user/balance`。

## curl 示例

```bash
curl -L -X GET "https://api.deepseek.com/user/balance" \
  -H "Accept: application/json" \
  -H "Authorization: Bearer ${DEEPSEEK_API_KEY}"
```

## 成功响应

```json
{
  "is_available": true,
  "balance_infos": [
    {
      "currency": "CNY",
      "total_balance": "110.00",
      "granted_balance": "10.00",
      "topped_up_balance": "100.00"
    }
  ]
}
```

字段说明：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `is_available` | `boolean` | 当前账户是否有余额可供 API 调用 |
| `balance_infos` | `object[]` | 分币种余额列表 |
| `balance_infos[].currency` | `string` | 币种，可能是 `CNY` 或 `USD` |
| `balance_infos[].total_balance` | `string` | 总可用余额，包含赠金和充值余额 |
| `balance_infos[].granted_balance` | `string` | 未过期赠金余额 |
| `balance_infos[].topped_up_balance` | `string` | 充值余额 |

## RetainPDF 接入建议

- Token 连通性检测可以先调用 `/v1/models` 判断 Key 是否有效，再调用 `/user/balance` 判断余额是否可用。
- 如果 `is_available=false` 或接口返回 `402`，前端应提示“DeepSeek 余额不足”，不要只显示“Key 无效”。
- 余额接口只适用于 DeepSeek 官方 API；如果用户配置了 OpenRouter、硅基流动或其他兼容服务，不应调用这个接口。

参考：DeepSeek 官方文档 `Get User Balance / 查询余额`。
